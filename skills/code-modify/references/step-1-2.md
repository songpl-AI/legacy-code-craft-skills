# Step 1-2: 读取脚本 + 生成风险报告

## Step 1 — 读取并理解脚本

1. 完整读取目标脚本
2. 理解脚本结构和现有业务逻辑
3. 识别需要修改/新增的位置
4. **有歧义或不确定的地方，必须停下来问用户**，不要猜测

---

## Step 2 — 生成风险报告

在修改代码前，**必须**输出以下风险报告：

```
## 风险报告

> **报告类型**：修改风险评估 | **生成时间**：YYYY-MM-DD

### 修改范围
- 涉及函数：xxx, yyy
- 新增函数：zzz
- 修改已有代码：[是/否] [如果是，列出具体位置]

### 影响分析
- 可能影响的功能：xxx
- Watch/Fire 事件依赖链：EVENT_A → 脚本B, 脚本C
- HTTP 接口依赖：/v3/xxx

### 风险等级：[低/中/高]
- **低**：影响 <3 个函数，无事件依赖，纯新增代码
- **中**：影响 3-10 个函数，或有 1-2 个事件依赖，或少量修改
- **高**：影响 >10 个函数，或修改核心流程，或 3+ 事件依赖

### 风险说明
[具体描述可能的风险点]
```

**用户确认后才能继续**
