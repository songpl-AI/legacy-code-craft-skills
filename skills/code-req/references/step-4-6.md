# Step 4-6: 输出路径 & 生成报告 & 写入文件

> **路径约定**：
> - `{AGENT_DIR}` = `.claude`
> - `{baseDir}` = `project-data`
> - **完整输出路径** = `.claude/project-data/analysis/{功能名}_{日期}/`

## Step 4 — 提取需求名和输出路径

- 从文档标题或首段提取功能名
- 输出目录：`{AGENT_DIR}/{baseDir}/analysis/{功能名}_{YYYY-MM-DD}/`
- 输出两个文件：`需求分析.md`（完整分析）和 `待确认问题.md`（可直接发出）

## Step 6 — 写入文件并输出摘要

1. 写入 `{AGENT_DIR}/{baseDir}/analysis/{功能名}_{YYYY-MM-DD}/需求分析.md`
2. 写入 `{AGENT_DIR}/{baseDir}/analysis/{功能名}_{YYYY-MM-DD}/待确认问题.md`
3. 输出：

```
✅ 需求分析完成
📁 {AGENT_DIR}/{baseDir}/analysis/{功能名}_{日期}/
   ├── 需求分析.md       （完整分析，内部用）
   └── 待确认问题.md     （可直接发给策划/后端）

📋 摘要：
- 功能点：客户端 X 项（🆕X / 🔧X / ✅X），服务器 X 项
- 工作量估算：客户端约 X 天，服务器约 X 天
- 涉及脚本：修改 X 个，新增 X 个
- 风险：🔴 X / 🟡 X / 🟢 X
- 待确认问题：X 项（其中 X 项阻塞开发）
```
