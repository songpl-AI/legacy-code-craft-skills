# 与 code-modify 集成

## 关联提示机制

### 触发时机

code-modify 执行时（Step 1 读取脚本后），自动检查：

1. 读取 `{AGENT_DIR}/{baseDir}/memory/common/index.json`
2. 遍历 entries，检查模块名是否出现在脚本注释或函数名中
3. 有关联时，输出提示：

```
⚠️ 公共知识提醒

该脚本可能涉及 [协议层规范]，memory/common/协议层规范.md 中有相关记录：

> HTTP 请求必须先检查 magic number，否则可能接收到恶意数据

是否查看详情？(y/n)
```

### 使用场景

| 场景 | 提示知识 |
|------|---------|
| 修改网络相关代码前 | 网络重连机制 |
| 修改状态机相关代码前 | 状态机设计规范 |
| 修改协议解析相关代码前 | 协议层规范 |

## code-modify 调用时机

code-modify 在以下时机调用 code-common：

| 时机 | 集成方式 |
|------|---------|
| Step 1 读取脚本后 | 自动触发关联检查 |
| Step 6 输出修改说明后 | 可选提示更新公共知识 |

code-modify 的 SKILL.md 中已定义集成点，实际调用通过框架自动完成。
