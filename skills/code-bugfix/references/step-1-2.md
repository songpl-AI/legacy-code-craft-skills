# Step 1-2: 解析 Bug 报告 & 关联开发记录

## Step 1 — 解析 Bug 报告

读取 Bug 报告，提取以下字段：
- `tag`：功能模块 TAG（最关键，用于关联归档）
- `feature`：功能名称
- `severity`：严重程度
- `操作路径`：复现步骤
- `实际行为`：问题现象
- `日志`：真机日志原文
- `日志扫描摘要`：初步分析（如有，优先利用，避免重复扫描）

**如果 tag 缺失**：
```
Bug 报告中缺少功能模块 TAG，无法自动定位代码。
请告诉我这个 bug 属于哪个功能模块？
当前已归档的模块：（读取 script-index.md 列出）
```

---

## Step 2 — 关联开发记录（三级降级策略）

按以下顺序逐级尝试，信息越多定位越准：

### 级别 1：查归档（最精准）

读取 `{AGENT_DIR}/{baseDir}/archive/scripts/{TAG}/` 目录：
- 有归档 → 获取功能说明、涉及函数、HTTP 接口、**修改历史**（关键！）
- 无归档 → 注明「⚠️ 该功能无归档，建议修复后执行 `/code-archive create`」

### 级别 2：查脚本索引（兜底定位）

读取 `{AGENT_DIR}/{baseDir}/script-index.md`，用 `tag` 匹配：
- 命中 → 获取脚本文件名和功能定位描述
- 未命中 → 注明「⚠️ TAG 不在索引中」

### 级别 3：全量 Grep（最后手段）

如果前两级都未命中，提取关键词在 `scripts/` 和 `remoteScripts/` 全量搜索：
- 关键词：`feature` 字段、操作路径中的功能描述词、日志中出现的函数名
- 找到候选脚本后，输出「通过关键词搜索找到候选脚本，请确认是否正确」

### 输出关联摘要

**必须标注数据来源级别**：
```
## 关联的开发记录

- **数据来源**：🟢 归档（精准）/ 🟡 索引（兜底）/ 🔴 Grep搜索（候选，请确认）
- **脚本**：scripts/script_name.lua
- **归档功能**：功能名称 / ⚠️ 未归档
- **核心函数**：InitUI(), OnEvent()
- **相关事件**：{{EVENT_NAME}}（Watch）
- **最近修改**：YYYY-MM-DD 开发者 - 修改描述
- **关联接口**：/v3/module/api
```
