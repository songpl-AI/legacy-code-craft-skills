# 查找脚本时的回退策略

当用户描述功能需求（如"找助力相关代码"）而**不是**直接提供脚本路径时，按以下顺序查找：

---

## 第一步：查 script-index.md 索引表

- 先按 **TAG 列**匹配关键词
- 再按**功能定位**、**关键功能**列匹配
- 找到 → 通过脚本文件列定位文件路径，进入分析流程

---

## 第二步：索引未命中 → 全量 Grep

```
在 scripts/ 和 remoteScripts/ 目录下，用关键词搜索所有 .lua 文件
例如：搜索 "boost-map" / "助力" / "like" 等业务关键词
```

---

## 第三步：找到脚本后

- 执行完整分析（Step 1~3）
- 执行 Step 4~8 将脚本 TAG 和事件写入索引，下次直接命中

---

## 大文件处理

如果脚本超过 1000 行：
1. 先使用 Grep 提取关键函数列表
2. 逐函数分析，而不是一次读取全部
3. 报告中只保留核心信息，省略次要细节
