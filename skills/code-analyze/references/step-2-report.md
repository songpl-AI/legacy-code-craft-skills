### UI 节点映射表

> 所有在 `InitUI()` 或 `initialize()` 中通过 `Find()` 绑定的节点

| 变量名 | Find 路径 | 类型 | 用途 |
|--------|---------|------|------|
| `self.btnClose` | `top/Btn_Close` | Button | 关闭面板 |
| `self.txtTitle` | `top/Txt_Title` | Text | 标题文本 |

**循环节点：**
| 数组变量 | 路径规律 | 数量 | 用途 |
|---------|---------|------|------|
| `self.itemNodes[1~9]` | `list/Item_00~08` | 9 | 列表 item |

---

### HTTP 接口列表

| 接口 | 方法 | 函数 | 入参 | 返回字段 | 有无 fail 回调 |
|------|------|------|------|---------|--------------|
| `/v3/module/api` | POST | `RequestXxx()` | `{key: val}` | `data.field` | ✅/❌ |

---

### Watch / Fire 事件列表

| 类型 | 事件名 | 触发函数 | 参数 | 说明 |
|------|--------|---------|------|------|
| Watch | `{{EVENT_NAME}}` | `OnXxx()` | `args[0].key` | 打开面板 |
| Fire | `{{EVENT_NAME}}` | `OnClick()` | `{key: val}` | 通知其他模块 |

---

### 全局状态依赖

| 变量 | 使用方式 | 所在函数 |
|------|---------|---------|
| `{{GLOBAL_CONFIG}}.FieldName` | 读取 | `OnEvent()` |

---

### 空结果说明规则

如果某个分析模块结果为空，必须给出原因：
- UI 节点为空 → "该脚本未通过 `Find()` 绑定 UI 节点（可能是纯逻辑脚本）"
- HTTP 接口为空 → "该脚本未发起 HTTP 请求（可能只处理 UI 或事件转发）"
- Watch/Fire 为空 → "该脚本未注册或触发任何 Watch/Fire 事件"

---

### 函数调用链（主流程）

```
initialize()
  └── InitUI()          → 绑定 UI 节点
  └── RegisterEvents()  → Watch 事件
  └── RequestXxx()      → 初始化数据请求
        └── RefreshUI() → 刷新界面

OnXxxClick()
  └── HttpRequest(...)
        └── code == 0 → RefreshXxx()
        └── code != 0 → ShowToast / {{LOG_ERROR}}
```

### 代码质量问题

| 行号 | 问题类型 | 描述 | 风险 |
|------|---------|------|------|
| 250 | 无 fail 回调 | HttpRequest 缺少网络错误处理 | 中 |
| 869 | 错误判断 | 用 `data.msg == "success"` 而非 `msg.code == 0` | 高 |

### 可插入点分析

> 告诉你在哪里可以安全地新增逻辑

| 位置 | 行号 | 适合添加 | 注意事项 |
|------|------|---------|---------|
| `OnEvent` 成功回调后 | ~220 | 成功后置逻辑 | 注意当前 code 判断逻辑 |
| `RefreshAll()` 末尾 | ~600 | 新状态刷新逻辑 | 此函数每次打开面板都会调用 |
| `initialize()` 末尾 | ~50 | 新 Watch 事件注册 | 确保 Exit() 中同步清理 |
